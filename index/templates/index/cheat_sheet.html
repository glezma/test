Git Commands

Initiate .git
git init
git config --list
Add changes to the local base
git add --all
Commit changes to the local repository adding a compulsory tag
git commit -am "This is a change"
Verify to which remote git repository I am connected
git remote -v
Push local commited content to the master branch of the remote origin of name 'origin_name'
git push origin_name master
sets (url for remoteorigin of name origin)
git remote set-url origin https://github.com/glezma/training-hub.git

Heroku Commands

It is a one off command, after the first execution it keeps logged in
heroku login
Creates an app with name training-hub
heroku create training-hub
Clone the .git from heroku repostoty called "training-hub"
heroku git:clone -a training-hub
Connecting remotely with project dry-adlands-897 (you must be inside the folder containing the .git file)
heroku git:remote -a dry-adlands-8976
Deploy to heroku server
git push heroku master
Open the heroku app to which we are connected
heroku open
More Heroku commands
heroku addons
Create a heroku postgress addon and attach it to the app
heroku addons:create heroku-postgresql:hobby-dev
addons:create heroku-postgresql:
Esperar que la base de datos aparezca online

Native Python- Pip and Virtual Environment

Create a virtual env
cd $ENV_BASE_DIR; virtualenv $ENVIRONMENT_NAME
Activate an environment
source $ENV_BASE_DIR/$ENVIRONMENT_NAME/bin/activate
Deactivate current virtual env
deactivate
Instalation of a python module
pip install $PACKAGE_NAME
See the installed modules
pip freeze
Generates a txt with a list of installated modules
pip freeze > requirements.txt
Install several python modules from txt list created by pip freeze
pip install -r requirements.txt

Conda

Create a virtual env (--file req.txt opcional). Los venv se generan en la carpeta de anaconda (e.g. C:\Anaconda3\envs)
conda create -n name_environment --file req.txt
conda create -n py33 python=3.3 anaconda Activate an environment
activate name_environment
Deactivate current virtual env
deactivate
Instalacion de un modulo python se puede instalar con pip (conda registra los modulos) si no esta en conda buscar en https://conda.anaconda.org/binstar
conda install $PACKAGE_NAME
Ver los modulos instalados
conda list
Generacion de txt con los modulos instalados en el venv actual
conda list -e > req.txt
Instalacion de varios modulos python en un txt
nunca usado

Environment variables

Create a temporal environment variable in Mac
export APP_SETTINGS="config.DevelopmentConfig"
export DATABASE_URL="postgresql://localhost/database"
Create a permanent environment variable in Mac (add the following to ~/.bash_profile)
export APP_SETTINGS=config.DevelopmentConfig
export DATABASE_URL=postgresql://localhost/database
Format for postgress URL
postgres://gkdpsxdewddoqc:sEAz2-G7h64myUrxqAHzhCltle@ec2-54-197-241-239.compute-1.amazonaws.com:5432/d7s0v25a53k5rt
postgresql://[user[:password]@][netloc][:port][/dbname][?param1=value1&...]
